Chapter 1: Introduction to Big Data Analytics in Cybersecurity

เนื้อหาครอบคลุม Data Analytics, Data Science และคุณลักษณะของ Big Data

ข้อ 1
Data Analytics กับ Data Science แตกต่างกันอย่างไร (อธิบายมาพอสังเขป)
✅ คำตอบ:
Data Analytics (การวิเคราะห์ข้อมูล): คือศาสตร์ของการวิเคราะห์ข้อมูลดิบ (Raw Data) เพื่อหาข้อสรุปจากข้อมูลนั้นๆ โดยมุ่งเน้นการใช้สถิติเชิงพรรณนาพื้นฐาน (Basic Descriptive Statistics) สร้างภาพข้อมูล (Data Visualization) และสื่อสารผลลัพธ์เพื่อให้เกิดความเข้าใจและนำไปสู่การตัดสินใจทางธุรกิจ นักวิเคราะห์ข้อมูล (Data Analyst) จะทำงานกับข้อมูลที่มีโครงสร้าง (Structured Data) เป็นหลัก เพื่อค้นหาข้อมูลเชิงลึก (Business Insight) สำหรับการตัดสินใจและวางกลยุทธ์ Data Science (วิทยาศาสตร์ข้อมูล): เป็นศาสตร์ที่กว้างกว่า โดยครอบคลุมกระบวนการทั้งหมดตั้งแต่การจัดการกับข้อมูลขนาดใหญ่ (Big Data) ซึ่งรวมถึงการทำความสะอาดข้อมูล (Data Cleansing) การเตรียมข้อมูล (Preparation) และการวิเคราะห์ นักวิทยาศาสตร์ข้อมูล (Data Scientist) จะรวบรวมข้อมูลจากหลายแหล่งและใช้เทคนิคขั้นสูง เช่น Machine Learning, Predictive Analytics และ Sentiment Analysis เพื่อสกัดข้อมูลที่สำคัญออกมา นอกจากนี้ยังทำงานได้ทั้งกับข้อมูลที่มีโครงสร้าง (Structured Data) และไม่มีโครงสร้าง (Unstructured Data) เพื่อพัฒนาโมเดลและเครื่องมือที่ช่วยตอบโจทย์ทางธุรกิจ

ข้อ 2
อธิบายประเภทการวิเคราะห์ข้อมูล (Types of Data Analytics) ทั้ง 4 ระดับ มาพอสังเขป
✅ คำตอบ:
ประเภทการวิเคราะห์ข้อมูลมี 4 ระดับ ดังนี้ 1. Descriptive Analytics (การวิเคราะห์เชิงพรรณนา): เป็นการวิเคราะห์เพื่อตอบคำถามว่า "เกิดอะไรขึ้น" (What happened?) โดยจะสรุปข้อมูลในอดีตให้อยู่ในรูปแบบที่เข้าใจง่าย เช่น การทำรายงานสรุปยอดขาย, แดชบอร์ดแสดงผลข้อมูล 2. Diagnostic Analytics (การวิเคราะห์เชิงวินิจฉัย): เป็นการวิเคราะห์เพื่อตอบคำถามว่า "ทำไมสิ่งนั้นถึงเกิดขึ้น" (Why did it happen?) เป็นการเจาะลึกลงไปในข้อมูลเพื่อหาสาเหตุของเหตุการณ์ที่เกิดขึ้น เช่น การวิเคราะห์หาสาเหตุที่ทำให้ยอดขายลดลง 3. Predictive Analytics (การวิเคราะห์เชิงพยากรณ์): เป็นการวิเคราะห์เพื่อตอบคำถามว่า "จะเกิดอะไรขึ้นในอนาคต" (What will happen?) โดยใช้ข้อมูลในอดีตมาสร้างแบบจำลองเพื่อทำนายแนวโน้มในอนาคต เช่น การพยากรณ์ยอดขายในไตรมาสถัดไป 4. Prescriptive Analytics (การวิเคราะห์เชิงให้คำแนะนำ): เป็นการวิเคราะห์เพื่อตอบคำถามว่า "ควรจะทำอย่างไร" (What should we do?) หรือ "ถ้าทำแบบนี้จะเกิดอะไรขึ้น" เป็นขั้นที่สูงที่สุด โดยระบบจะให้คำแนะนำทางเลือกที่ดีที่สุดเพื่อนำไปปฏิบัติจริง เช่น การแนะนำโปรโมชันที่เหมาะสมกับลูกค้าแต่ละรายเพื่อเพิ่มยอดขาย

ข้อ 3
คุณลักษณะสำคัญของ Big Data ทั้ง 5V ประกอบด้วยอะไร พร้อมอธิบายมาพอสังเขป
✅ คำตอบ:
คุณลักษณะสำคัญของ Big Data (5V's) ประกอบด้วย 1. Volume (ปริมาณ): ข้อมูลมีขนาดใหญ่มากในระดับ Terabytes ถึง Zettabytes ซึ่งท้าทายต่อการจัดเก็บและประมวลผลด้วยวิธีการแบบดั้งเดิม 2. Velocity (ความเร็ว): ข้อมูลถูกสร้างขึ้นและเปลี่ยนแปลงอย่างรวดเร็วแบบเรียลไทม์ (Real-time) หรือใกล้เคียงเรียลไทม์ ทำให้ต้องมีการประมวลผลที่รวดเร็วเพื่อให้ทันต่อสถานการณ์ 3. Variety (ความหลากหลาย): ข้อมูลมีรูปแบบที่หลากหลาย ทั้งข้อมูลที่มีโครงสร้าง (Structured Data) เช่น ข้อมูลในตาราง, กึ่งมีโครงสร้าง (Semi-structured Data) เช่น ไฟล์ XML, และไม่มีโครงสร้าง (Unstructured Data) เช่น ข้อความ, รูปภาพ, วิดีโอ 4. Veracity (ความถูกต้อง): ข้อมูลมีความไม่แน่นอน อาจมีความไม่สอดคล้องกัน ข้อมูลที่ไม่สมบูรณ์ หรือความคลุมเครือ ซึ่งต้องมีการตรวจสอบและยืนยันความถูกต้องของข้อมูลก่อนนำไปใช้ 5. Value (คุณค่า): ข้อมูลที่มีอยู่จะต้องสามารถนำมาวิเคราะห์เพื่อสร้างประโยชน์หรือคุณค่าให้กับองค์กรได้ เช่น การสร้างรายได้, การลดต้นทุน, หรือการปรับปรุงการบริการ

Chapter 2: Cybersecurity Framework & Cyber Attack Framework

เนื้อหาครอบคลุม NIST Cybersecurity Framework 2.0 และ Cyber Kill Chain

ข้อ 4
NIST Cybersecurity Framework 2.0 ประกอบด้วยฟังก์ชันหลักคืออะไร พร้อมอธิบายแต่ละฟังก์ชันของเฟรมเวิร์กมาพอสังเขป
✅ คำตอบ:
NIST Cybersecurity Framework (CSF) 2.0 ประกอบด้วย 6 ฟังก์ชันหลัก ดังนี้ 1. Govern (การกำกับดูแล): เป็นฟังก์ชันที่เพิ่มขึ้นมาใหม่ เน้นการสร้างและกำกับดูแลกลยุทธ์, ความคาดหวัง และนโยบายการจัดการความเสี่ยงด้านความมั่นคงปลอดภัยไซเบอร์ขององค์กร เพื่อให้สอดคล้องกับเป้าหมายทางธุรกิจโดยรวม 2. Identify (การระบุ): การทำความเข้าใจบริบททางธุรกิจ, ทรัพย์สิน (Assets) ที่สำคัญ และความเสี่ยงด้านความมั่นคงปลอดภัยไซเบอร์ที่เกี่ยวข้อง เพื่อใช้ในการบริหารจัดการความเสี่ยง 3. Protect (การป้องกัน): การใช้มาตรการป้องกันที่เหมาะสมเพื่อจำกัดหรือควบคุมผลกระทบจากเหตุการณ์ความมั่นคงปลอดภัยไซเบอร์ เพื่อให้การบริการที่สำคัญดำเนินไปได้อย่างต่อเนื่อง 4. Detect (การตรวจจับ): การพัฒนาและใช้กิจกรรมที่เหมาะสมเพื่อตรวจจับการเกิดขึ้นของเหตุการณ์ความมั่นคงปลอดภัยไซเบอร์ได้อย่างทันท่วงที 5. Respond (การตอบสนอง): การดำเนินการที่เหมาะสมเมื่อตรวจพบเหตุการณ์ความมั่นคงปลอดภัยไซเบอร์ เพื่อควบคุมผลกระทบของเหตุการณ์ 6. Recover (การฟื้นฟู): การวางแผนและดำเนินการเพื่อฟื้นฟูความสามารถและบริการที่ได้รับผลกระทบจากเหตุการณ์ความมั่นคงปลอดภัยไซเบอร์ให้กลับสู่ภาวะปกติ

ข้อ 5
ขั้นตอนทั้ง 5 ของการสร้าง Cybersecurity Framework (CSF) Organizational Profile ตามหลักการของ NIST ประกอบอะไร
✅ คำตอบ:
ขั้นตอนทั้ง 5 ของการสร้างและใช้ CSF Organizational Profile ประกอบด้วย 1. Scope the Organizational Profile (กำหนดขอบเขต): กำหนดขอบเขตของ Profile ว่าจะครอบคลุมทั้งองค์กรหรือเฉพาะส่วนงานใดส่วนงานหนึ่ง 2. Gather needed information (รวบรวมข้อมูลที่จำเป็น): รวบรวมข้อมูลที่เกี่ยวข้องกับกระบวนการ, นโยบาย, และแนวปฏิบัติในการรักษาความมั่นคงปลอดภัยไซเบอร์ในปัจจุบัน 3. Create the Organizational Profile (สร้าง Profile): สร้าง Profile ของสถานะปัจจุบัน (Current State) และสถานะที่ต้องการ (Target State) โดยเทียบกับ Categories และ Subcategories ใน Framework Core 4. Analyze gaps and create an action plan (วิเคราะห์ช่องว่างและสร้างแผนปฏิบัติการ): เปรียบเทียบระหว่าง Current Profile และ Target Profile เพื่อระบุช่องว่าง (Gaps) และจัดลำดับความสำคัญในการดำเนินการ พร้อมทั้งสร้างแผนปฏิบัติการ (Action Plan) เพื่อลดช่องว่างนั้น 5. Implement action plan and update Profile (ดำเนินการตามแผนและปรับปรุง Profile): นำแผนปฏิบัติการไปใช้ และปรับปรุง Profile อย่างสม่ำเสมอตามการเปลี่ยนแปลงขององค์กรและสภาพแวดล้อม

ข้อ 6
อธิบายแต่ละขั้นตอนใน Cyber Kill Chain framework มาพอสังเขป
✅ คำตอบ:
Cyber Kill Chain framework มี 7 ขั้นตอนที่ผู้โจมตีต้องดำเนินการเพื่อให้บรรลุเป้าหมาย ดังนี้ 1. Reconnaissance (การลาดตระเวน): ผู้โจมตีรวบรวมข้อมูลเกี่ยวกับเป้าหมาย เช่น ที่อยู่อีเมล, ข้อมูลองค์กร, ช่องโหว่ของระบบ เพื่อวางแผนการโจมตี 2. Weaponization (การสร้างอาวุธ): ผู้โจมตีสร้าง "อาวุธ" หรือมัลแวร์ (Malware) โดยใช้ช่องโหว่ (Exploit) ที่พบร่วมกับ Backdoor เพื่อสร้าง Payload ที่สามารถส่งไปยังเป้าหมายได้ 3. Delivery (การนำส่ง): ผู้โจมตีส่งอาวุธที่สร้างขึ้นไปยังเป้าหมายผ่านช่องทางต่างๆ เช่น อีเมล (Phishing), เว็บไซต์, หรือ USB Drive 4. Exploitation (การใช้ช่องโหว่โจมตี): เมื่ออาวุธถูกส่งถึงเป้าหมายและถูกเปิดใช้งาน มันจะใช้ประโยชน์จากช่องโหว่ของซอฟต์แวร์หรือระบบปฏิบัติการเพื่อรันโค้ดของผู้โจมตี 5. Installation (การติดตั้ง): มัลแวร์จะทำการติดตั้งตัวเองลงในเครื่องของเหยื่อ เพื่อให้สามารถคงอยู่และทำงานในระบบต่อไปได้แม้จะมีการรีสตาร์ทเครื่อง 6. Command & Control (C2) (การสั่งการและควบคุม): มัลแวร์จะสร้างช่องทางการสื่อสารกลับไปยังเซิร์ฟเวอร์ของผู้โจมตี เพื่อรอรับคำสั่งและควบคุมเครื่องของเหยื่อจากระยะไกล 7. Actions on Objectives (การดำเนินการตามเป้าหมาย): ผู้โจมตีลงมือดำเนินการตามเป้าหมายที่วางไว้ เช่น การขโมยข้อมูล (Data Exfiltration), การทำลายข้อมูล, หรือการเข้ารหัสข้อมูลเพื่อเรียกค่าไถ่ (Ransomware)

Chapter 3: Cybersecurity Threat and Risks and Awareness

เนื้อหาครอบคลุม Information Security, Cybersecurity และภัยคุกคามต่างๆ

ข้อ 7
อธิบายความแตกต่างระหว่าง Information Security และ Cybersecurity
✅ คำตอบ:
Information Security (ความมั่นคงปลอดภัยสารสนเทศ): มุ่งเน้นการปกป้อง "สารสนเทศ" หรือ "ข้อมูล" ในทุกรูปแบบ ไม่ว่าจะเป็นข้อมูลดิจิทัล (Digital) หรือข้อมูลกายภาพ (Analog) เช่น เอกสารกระดาษ โดยให้ความสำคัญกับหลักการ 3 ประการคือ การรักษาความลับ (Confidentiality), ความถูกต้องครบถ้วน (Integrity), และความพร้อมใช้งาน (Availability) หรือที่เรียกว่า CIA Triad Cybersecurity (ความมั่นคงปลอดภัยไซเบอร์): เป็นส่วนหนึ่งของ Information Security แต่มุ่งเน้นการปกป้อง "สินทรัพย์ในโลกไซเบอร์" (Cyberspace) จากการถูกโจมตีทางดิจิทัล ซึ่งรวมถึงการปกป้องระบบคอมพิวเตอร์, เครือข่าย, โปรแกรม และข้อมูลที่จัดเก็บและส่งผ่านเทคโนโลยีสารสนเทศและการสื่อสาร (ICT) สรุป: Information Security มีขอบเขตที่กว้างกว่า ครอบคลุมข้อมูลทุกรูปแบบ ในขณะที่ Cybersecurity จะเน้นไปที่การป้องกันภัยคุกคามในโลกดิจิทัลโดยเฉพาะ

ข้อ 8
อธิบาย Deepfake Attack คืออะไร พร้อมยกตัวอย่างการโจมตีลักษณะนี้มา 1 ตัวอย่างและแนวทางการป้องกัน
✅ คำตอบ:
Deepfake Attack คือ: การใช้เทคโนโลยีปัญญาประดิษฐ์ (AI) โดยเฉพาะเทคนิค Deep Learning ในการสร้างสื่อสังเคราะห์ (Synthetic Media) ไม่ว่าจะเป็นวิดีโอ, รูปภาพ หรือเสียง ที่ปลอมแปลงใบหน้าหรือเสียงของบุคคลอื่นให้ดูสมจริงอย่างยิ่ง เพื่อใช้ในการหลอกลวง, สร้างความเสื่อมเสีย, เผยแพร่ข่าวปลอม หรือโจมตีทางไซเบอร์ ตัวอย่างการโจมตี: มีการใช้เทคโนโลยี Deepfake ปลอมแปลงวิดีโอและเสียงของบุคคลที่มีชื่อเสียง เช่น อีลอน มัสก์ (Elon Musk) โดยสร้างวิดีโอไลฟ์สดปลอมที่ดูเหมือนว่าอีลอน มัสก์ กำลังโปรโมตการลงทุนในสกุลเงินดิจิทัล และชักชวนให้ผู้ชมสแกน QR Code เพื่อโอนเงินเข้าร่วมลงทุน ซึ่งแท้จริงแล้วเป็นการหลอกลวงเพื่อขโมยเงินจากเหยื่อ แนวทางการป้องกัน: • ตั้งข้อสงสัย (Be Skeptical): หากพบบุคคลสาธารณะหรือคนรู้จักขอข้อมูลส่วนตัว, ขอให้โอนเงิน หรือเสนอสิ่งที่ดูดีเกินจริงผ่านช่องทางออนไลน์ ให้ตั้งข้อสงสัยไว้ก่อนเสมอ • ตรวจสอบความผิดปกติ: สังเกตความไม่สมจริงของวิดีโอ เช่น การกะพริบตาที่ไม่เป็นธรรมชาติ, สีผิวที่ไม่สม่ำเสมอ, การขยับปากที่ไม่ตรงกับเสียง, หรือเสียงที่เหมือนหุ่นยนต์ • ยืนยันตัวตนผ่านช่องทางอื่น (Verify Identity): หากเป็นคนรู้จักที่ติดต่อมาเพื่อขอความช่วยเหลือทางการเงิน ควรโทรศัพท์หรือวิดีโอคอลกลับไปเพื่อยืนยันว่าเป็นบุคคลนั้นจริง ไม่ควรเชื่อจากการสนทนาผ่านข้อความหรือเสียงที่ถูกส่งมาเพียงอย่างเดียว • ติดตามข่าวสาร: รับรู้ข่าวสารเกี่ยวกับเทคนิคการหลอกลวงใหม่ๆ อยู่เสมอ เพื่อให้รู้เท่าทันกลโกงของมิจฉาชีพ

ข้อ 9
Spyware-Applications และ Adware-Applications แตกต่างกันอย่างไร
✅ คำตอบ:
Spyware-Applications (สปายแวร์): เป็นแอปพลิเคชันที่ถูกออกแบบมาเพื่อ "สอดแนม" หรือติดตามกิจกรรมของผู้ใช้ โดยไม่ได้รับความยินยอมอย่างชัดแจ้งจากผู้ใช้ มักจะถูกติดตั้งแฝงมากับโปรแกรมอื่น มีวัตถุประสงค์เพื่อขโมยข้อมูลส่วนบุคคล, รหัสผ่าน, หรือข้อมูลทางการเงิน Adware-Applications (แอดแวร์): เป็นแอปพลิเคชันที่แสดงโฆษณาที่ไม่พึงประสงค์แก่ผู้ใช้ โดยทั่วไปแล้วผู้ใช้อาจจะ "ยินยอม" ให้ติดตั้งโดยไม่รู้ตัว (มักจะพ่วงมากับซอฟต์แวร์ฟรี) โดย Adware จะติดตามพฤติกรรมการใช้งานอินเทอร์เน็ตของผู้ใช้เพื่อนำเสนอโฆษณาที่ตรงเป้าหมาย แม้จะไม่เป็นอันตรายเท่าสปายแวร์ แต่ก็สร้างความรำคาญและอาจทำให้ประสิทธิภาพของเครื่องลดลง ความแตกต่างที่สำคัญ: คือ "ความยินยอม" (Permission) โดย Spyware จะทำงานโดยที่ผู้ใช้ไม่รู้และไม่ยินยอม ในขณะที่ Adware อาจถือว่าผู้ใช้ได้ให้ความยินยอม (แม้จะไม่ตั้งใจ) ในตอนติดตั้งโปรแกรม

Chapter 4: Incident Management and Cyber Resilience Framework & Log Management

เนื้อหาครอบคลุมการจัดการเหตุการณ์และการจัดการบันทึกเหตุการณ์

ข้อ 10
อธิบายวงจรการตอบสนองเหตุการณ์ (Cyber Incident Response Cycle) มาพอสังเขป
✅ คำตอบ:
วงจรการตอบสนองเหตุการณ์ (Cyber Incident Response Cycle) ตามแนวทางของ NIST SP 800-61 ประกอบด้วย 4 ขั้นตอนหลัก ดังนี้ 1. Preparation (การเตรียมความพร้อม): เป็นขั้นตอนการวางแผนและเตรียมการก่อนเกิดเหตุการณ์จริง รวมถึงการจัดตั้งทีมนโยบาย, เครื่องมือ, และกระบวนการต่างๆ เพื่อให้พร้อมรับมือกับเหตุการณ์ได้อย่างมีประสิทธิภาพ 2. Detection & Analysis (การตรวจจับและวิเคราะห์): เป็นขั้นตอนการเฝ้าระวัง, ตรวจจับสัญญาณของเหตุการณ์ไม่พึงประสงค์, วิเคราะห์ข้อมูลเพื่อยืนยันว่าเป็นเหตุการณ์จริง, ประเมินขอบเขตและผลกระทบ, และจัดลำดับความสำคัญของเหตุการณ์ 3. Containment, Eradication & Recovery (การจำกัดขอบเขต, การกำจัด, และการกู้คืน): • Containment: จำกัดความเสียหายไม่ให้แพร่กระจายไปยังส่วนอื่นๆ ของระบบ • Eradication: กำจัดต้นตอของปัญหา เช่น มัลแวร์, ช่องโหว่ ออกจากระบบ • Recovery: กู้คืนระบบที่ได้รับผลกระทบให้กลับมาทำงานได้ตามปกติและปลอดภัย 4. Post-incident Activity (กิจกรรมหลังเกิดเหตุ): เป็นขั้นตอนการเรียนรู้จากเหตุการณ์ที่เกิดขึ้น จัดทำรายงานสรุป, วิเคราะห์สิ่งที่ทำได้ดีและสิ่งที่ควรปรับปรุง (Lessons Learned) และนำข้อมูลไปพัฒนากระบวนการเตรียมความพร้อมให้ดียิ่งขึ้นในอนาคต

ข้อ 11
อธิบายองค์ประกอบการจัดการบันทึกเหตุการณ์ (Log Management) ทั้ง 4 มาพอสังเขป
✅ คำตอบ:
องค์ประกอบของการจัดการบันทึกเหตุการณ์ (Log Management) มี 4 ส่วนหลัก ดังนี้ 1. Log Generation (การสร้าง Log): กำหนดว่าระบบหรืออุปกรณ์ใดบ้าง (เช่น เซิร์ฟเวอร์, ไฟร์วอลล์, แอปพลิเคชัน) ที่จำเป็นต้องเปิดใช้งานการบันทึกเหตุการณ์ และต้องบันทึกข้อมูลอะไรบ้าง 2. Log Transmission (การส่ง Log): กำหนดวิธีการและช่องทางในการส่งข้อมูล Log จากแหล่งต่างๆ มารวบรวมไว้ที่ส่วนกลาง (Centralized Log Server) อย่างปลอดภัย เพื่อป้องกันการสูญหายหรือการเปลี่ยนแปลงแก้ไขข้อมูลระหว่างทาง 3. Log Storage and Disposal (การจัดเก็บและทำลาย Log): กำหนดนโยบายในการจัดเก็บ Log เช่น ระยะเวลาที่ต้องเก็บรักษา, วิธีการจัดเก็บเพื่อรักษาความลับ (Confidentiality) และความถูกต้อง (Integrity) ของข้อมูล และกระบวนการทำลาย Log เมื่อครบกำหนดอายุการจัดเก็บ 4. Log Analysis (การวิเคราะห์ Log): กำหนดความถี่และวิธีการในการตรวจสอบและวิเคราะห์ข้อมูล Log เพื่อตรวจหาสิ่งผิดปกติ, เหตุการณ์ด้านความปลอดภัย, หรือการละเมิดนโยบาย รวมถึงกำหนดสิทธิ์ในการเข้าถึงข้อมูล Log ของผู้ที่เกี่ยวข้อง

Chapter 5: Intrusion Detection and Prevention Systems

เนื้อหาครอบคลุม IDS/IPS และวิธีการตรวจจับการบุกรุก

ข้อ 12
อธิบาย 3 ฟังก์ชั่นหลักของ Intrusion Prevention Systems (IPS) พร้อมยกตัวอย่างการทำงานของทั้ง 3 ฟังก์ชั่น
✅ คำตอบ:
3 ฟังก์ชันหลักของ Intrusion Prevention Systems (IPS) คือ 1. หยุดการโจมตีโดยตรง (Stops the attack itself): IPS สามารถยับยั้งการโจมตีที่ตรวจพบได้ทันที ตัวอย่าง: เมื่อ IPS ตรวจพบการรับส่งข้อมูลที่ตรงกับรูปแบบของมัลแวร์ที่รู้จัก มันจะทำการตัดการเชื่อมต่อเครือข่าย (Terminate the network connection) ของผู้โจมตีทันที เพื่อไม่ให้มัลแวร์สามารถแพร่กระจายหรือส่งข้อมูลออกไปได้ 2. เปลี่ยนแปลงสภาพแวดล้อมความปลอดภัย (Changes the security environment): IPS สามารถปรับเปลี่ยนการตั้งค่าของอุปกรณ์ความปลอดภัยอื่นๆ เพื่อรับมือกับการโจมตี ตัวอย่าง: เมื่อ IPS ตรวจพบความพยายามสแกนพอร์ตจาก IP Address หนึ่งซ้ำๆ กัน มันสามารถส่งคำสั่งไปให้ Firewall เพื่อสร้างกฎ (Rule) ใหม่ในการบล็อกการเชื่อมต่อทั้งหมดจาก IP Address นั้นชั่วคราว 3. เปลี่ยนแปลงเนื้อหาของการโจมตี (Changes the attack's content): IPS สามารถแก้ไขหรือลบส่วนที่เป็นอันตรายออกจากการรับส่งข้อมูล ทำให้ข้อมูลที่เหลือไม่เป็นอันตรายอีกต่อไป ตัวอย่าง: เมื่อมีอีเมลเข้ามาพร้อมไฟล์แนบที่ติดไวรัส IPS สามารถลบไฟล์แนบที่เป็นอันตรายนั้นออกไป แล้วจึงส่งต่อเฉพาะเนื้อหาอีเมลที่ปลอดภัยไปยังผู้รับ

ข้อ 13
อธิบายวิธีการตรวจจับ (Detection Methodologies) ทั้ง 3 ลักษณะ มาพอสังเขป อันได้แก่ Signature-based Detection, Anomaly-based Detection และ Stateful Protocol Analysis Detection
✅ คำตอบ:
วิธีการตรวจจับ 3 ลักษณะ ได้แก่ 1. Signature-based Detection (การตรวจจับตามลายเซ็น): เป็นการตรวจจับโดยเปรียบเทียบข้อมูลที่ตรวจสอบ (เช่น แพ็กเก็ตเครือข่าย, ไฟล์) กับฐานข้อมูลของ "ลายเซ็น" (Signatures) หรือรูปแบบ (Patterns) ของการโจมตีที่รู้จักอยู่แล้ว หากข้อมูลตรงกับลายเซ็นใดๆ ในฐานข้อมูล ระบบจะแจ้งเตือนว่าเป็นการโจมตี วิธีนี้มีประสิทธิภาพสูงในการตรวจจับภัยคุกคามที่รู้จักแล้ว แต่ไม่สามารถตรวจจับการโจมตีรูปแบบใหม่ๆ ที่ยังไม่มีลายเซ็นได้ 2. Anomaly-based Detection (การตรวจจับความผิดปกติ): เป็นการตรวจจับโดยสร้าง "โปรไฟล์" หรือแบบจำลองของพฤติกรรมที่ "ปกติ" ของระบบหรือเครือข่ายขึ้นมาก่อน จากนั้นระบบจะเฝ้าสังเกตพฤติกรรมปัจจุบัน หากมีพฤติกรรมใดที่เบี่ยงเบนไปจากโปรไฟล์ปกติอย่างมีนัยสำคัญ ระบบจะถือว่าเป็นความผิดปกติ (Anomaly) และแจ้งเตือน วิธีนี้สามารถตรวจจับการโจมตีรูปแบบใหม่ๆ (Zero-day attacks) ได้ แต่ก็อาจเกิด False Positives (การแจ้งเตือนที่ผิดพลาด) ได้หากพฤติกรรมปกติมีการเปลี่ยนแปลง 3. Stateful Protocol Analysis Detection (การตรวจจับโดยวิเคราะห์สถานะของโปรโตคอล): เป็นการตรวจจับโดยอาศัยความเข้าใจในสถานะ (State) และกฎของโปรโตคอลต่างๆ (เช่น HTTP, FTP) ระบบจะตรวจสอบว่าการสื่อสารเป็นไปตามมาตรฐานของโปรโตคอลนั้นๆ หรือไม่ หากมีการใช้งานคำสั่งหรือลำดับขั้นที่ผิดไปจากที่โปรโตคอลกำหนดไว้ จะถือว่าเป็นสิ่งผิดปกติ วิธีนี้มีความแม่นยำสูงในการตรวจจับการโจมตีที่ใช้ประโยชน์จากช่องโหว่ของโปรโตคอล แต่ไม่สามารถตรวจจับการโจมตีที่ไม่ได้ละเมิดกฎของโปรโตคอลได้

องค์ประกอบของ IDPS ประกอบด้วยอะไร พร้อมอธิบายหน้าที่ของแต่ละองค์ประกอบ มาพอสังเขป
✅ คำตอบ:
องค์ประกอบของระบบตรวจจับและป้องกันการบุกรุก (IDPS) โดยทั่วไปประกอบด้วย 4 ส่วนหลัก ดังนี้ 1. Sensor or Agent (เซ็นเซอร์หรือเอเจนต์): เป็นส่วนที่ทำหน้าที่เฝ้าระวังและรวบรวมข้อมูลเหตุการณ์ต่างๆ ที่เกิดขึ้น • Sensor: มักใช้กับระบบที่ตรวจสอบเครือข่าย (Network-based IDPS) โดยจะดักจับและวิเคราะห์แพ็กเก็ตข้อมูลบนเครือข่าย • Agent: เป็นซอฟต์แวร์ที่ติดตั้งบนเครื่องคอมพิวเตอร์แต่ละเครื่อง (Host-based IDPS) เพื่อตรวจสอบกิจกรรมภายในเครื่องนั้นๆ เช่น การเข้าถึงไฟล์, การทำงานของโปรเซส 2. Management Server (เซิร์ฟเวอร์จัดการ): เป็นศูนย์กลางที่รับข้อมูลเหตุการณ์จาก Sensor หรือ Agent หลายๆ ตัวมาประมวลผลและวิเคราะห์ในภาพรวม สามารถทำ Correlation คือการเชื่อมโยงเหตุการณ์จากหลายแหล่งที่มาเพื่อระบุการโจมตีที่ซับซ้อนซึ่ง Sensor แต่ละตัวอาจมองไม่เห็น 3. Database Server (เซิร์ฟเวอร์ฐานข้อมูล): เป็นที่สำหรับจัดเก็บข้อมูลเหตุการณ์ (Event Information) ที่ถูกบันทึกโดย Sensor, Agent, หรือ Management Server เพื่อใช้ในการวิเคราะห์ย้อนหลังหรือการทำ Forensic 4. Console (คอนโซล): เป็นส่วนติดต่อกับผู้ใช้งาน (User Interface) ที่ให้ผู้ดูแลระบบสามารถเข้ามาดูการแจ้งเตือน, วิเคราะห์เหตุการณ์, ตั้งค่าการทำงานของระบบ, และสร้างรายงานต่างๆ ได้
