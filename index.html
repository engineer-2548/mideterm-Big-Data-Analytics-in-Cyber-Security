ข้อ 1 - Chapter 1
Data Analytics กับ Data Science แตกต่างกันอย่างไร?

Data Analytics (การวิเคราะห์ข้อมูล):

  • ศาสตร์ของการวิเคราะห์ข้อมูลดิบ (Raw Data) เพื่อหาข้อสรุป
  • มุ่งเน้นการใช้สถิติเชิงพรรณนาพื้นฐาน และ Data Visualization
  • ทำงานกับข้อมูลที่มีโครงสร้าง (Structured Data) เป็นหลัก
  • เพื่อค้นหา Business Insight สำหรับการตัดสินใจ

Data Science (วิทยาศาสตร์ข้อมูล):

  • ศาสตร์ที่กว้างกว่า ครอบคลุมทั้งกระบวนการจัดการ Big Data
  • รวมถึง Data Cleansing, Preparation และการวิเคราะห์
  • ใช้เทคนิคขั้นสูง เช่น Machine Learning, Predictive Analytics
  • ทำงานได้ทั้ง Structured และ Unstructured Data

ข้อ 2 - Chapter 1
อธิบายประเภทการวิเคราะห์ข้อมูล (Types of Data Analytics) ทั้ง 4 ระดับ

1. Descriptive Analytics (การวิเคราะห์เชิงพรรณนา):

  • ตอบคำถาม "เกิดอะไรขึ้น" (What happened?)
  • สรุปข้อมูลในอดีตให้เข้าใจง่าย เช่น รายงานยอดขาย, แดชบอร์ด

2. Diagnostic Analytics (การวิเคราะห์เชิงวินิจฉัย):

  • ตอบคำถาม "ทำไมถึงเกิดขึ้น" (Why did it happen?)
  • เจาะลึกหาสาเหตุของเหตุการณ์

3. Predictive Analytics (การวิเคราะห์เชิงพยากรณ์):

  • ตอบคำถาม "จะเกิดอะไรขึ้น" (What will happen?)
  • ใช้ข้อมูลอดีตสร้างแบบจำลองทำนายอนาคต

4. Prescriptive Analytics (การวิเคราะห์เชิงให้คำแนะนำ):

  • ตอบคำถาม "ควรทำอย่างไร" (What should we do?)
  • ให้คำแนะนำทางเลือกที่ดีที่สุด

ข้อ 3 - Chapter 1
คุณลักษณะสำคัญของ Big Data ทั้ง 5V ประกอบด้วยอะไร?

5V's ของ Big Data:

  • Volume (ปริมาณ): ข้อมูลขนาดใหญ่ระดับ TB ถึง ZB
  • Velocity (ความเร็ว): ข้อมูลถูกสร้างและเปลี่ยนแปลงแบบ Real-time
  • Variety (ความหลากหลาย): ข้อมูลหลายรูปแบบ (Structured, Semi-structured, Unstructured)
  • Veracity (ความถูกต้อง): ความไม่แน่นอนและความไม่สมบูรณ์ของข้อมูล
  • Value (คุณค่า): ข้อมูลต้องสร้างประโยชน์ให้องค์กร

ข้อ 4 - Chapter 2
NIST Cybersecurity Framework 2.0 ประกอบด้วยฟังก์ชันหลักอะไรบ้าง?

6 ฟังก์ชันหลักของ NIST CSF 2.0:

  • Govern (การกำกับดูแล): สร้างกลยุทธ์และนโยบายความมั่นคงปลอดภัย
  • Identify (การระบุ): ทำความเข้าใจทรัพย์สินและความเสี่ยง
  • Protect (การป้องกัน): ใช้มาตรการป้องกันที่เหมาะสม
  • Detect (การตรวจจับ): ตรวจจับเหตุการณ์ความมั่นคงปลอดภัย
  • Respond (การตอบสนอง): ดำเนินการเมื่อตรวจพบเหตุการณ์
  • Recover (การฟื้นฟู): ฟื้นฟูความสามารถและบริการ

ข้อ 5 - Chapter 2
ขั้นตอนการสร้าง CSF Organizational Profile ตาม NIST มีอะไรบ้าง?

5 ขั้นตอนการสร้าง CSF Organizational Profile:

  • 1. Scope the Organizational Profile: กำหนดขอบเขตของ Profile
  • 2. Gather needed information: รวบรวมข้อมูลที่จำเป็น
  • 3. Create the Organizational Profile: สร้าง Current และ Target State
  • 4. Analyze gaps and create action plan: วิเคราะห์ช่องว่างและสร้างแผน
  • 5. Implement action plan and update Profile: ดำเนินการและปรับปรุง

ข้อ 6 - Chapter 2
อธิบายแต่ละขั้นตอนใน Cyber Kill Chain framework

7 ขั้นตอนของ Cyber Kill Chain:

  • 1. Reconnaissance: การลาดตระเวนรวบรวมข้อมูลเป้าหมาย
  • 2. Weaponization: สร้างมัลแวร์หรืออาวุธโจมตี
  • 3. Delivery: ส่งอาวุธไปยังเป้าหมาย (อีเมล, USB)
  • 4. Exploitation: ใช้ช่องโหว่โจมตีระบบ
  • 5. Installation: ติดตั้งมัลแวร์ในเครื่องเหยื่อ
  • 6. Command & Control (C2): สร้างช่องทางควบคุมระยะไกล
  • 7. Actions on Objectives: ดำเนินการตามเป้าหมาย (ขโมยข้อมูล, เรียกค่าไถ่)

ข้อ 7 - Chapter 3
ความแตกต่างระหว่าง Information Security และ Cybersecurity คืออะไร?

Information Security:

  • ปกป้อง "สารสนเทศ" ทุกรูปแบบ (Digital & Analog)
  • รวมถึงเอกสารกระดาษและข้อมูลกายภาพ
  • ยึดหลัก CIA Triad (Confidentiality, Integrity, Availability)
  • มีขอบเขตกว้างครอบคลุมข้อมูลทุกประเภท

Cybersecurity:

  • เป็นส่วนหนึ่งของ Information Security
  • มุ่งเน้นปกป้อง "สินทรัพย์ในโลกไซเบอร์"
  • ป้องกันการโจมตีทางดิจิทัล
  • ครอบคลุมระบบคอมพิวเตอร์, เครือข่าย, โปรแกรม

ข้อ 8 - Chapter 3
Deepfake Attack คืออะไร พร้อมตัวอย่างและแนวทางป้องกัน

Deepfake Attack: การใช้ AI/Deep Learning สร้างสื่อปลอมแปลง (วิดีโอ, เสียง, รูปภาพ) ที่ดูสมจริง

ตัวอย่างการโจมตี:

  • ปลอมวิดีโอบุคคลมีชื่อเสียง (เช่น Elon Musk) โปรโมตการลงทุนหลอกลวง
  • ใช้ชักชวนให้สแกน QR Code เพื่อโอนเงิน

แนวทางการป้องกัน:

  • ตั้งข้อสงสัย: ระวังคำขอที่ดูดีเกินจริง
  • ตรวจสอบความผิดปกติ: การกะพริบตา, สีผิว, การขยับปาก
  • ยืนยันตัวตน: โทรศัพท์หรือวิดีโอคอลกลับ
  • ติดตามข่าวสาร: รู้เท่าทันเทคนิคหลอกลวงใหม่ๆ

ข้อ 9 - Chapter 3
Spyware และ Adware แตกต่างกันอย่างไร?

Spyware-Applications:

  • ออกแบบมาเพื่อ "สอดแนม" ติดตามกิจกรรมผู้ใช้
  • ทำงานโดยไม่ได้รับความยินยอมชัดแจ้ง
  • ขโมยข้อมูลส่วนบุคคล, รหัสผ่าน, ข้อมูลการเงิน
  • ติดตั้งแฝงมากับโปรแกรมอื่น

Adware-Applications:

  • แสดงโฆษณาที่ไม่พึงประสงค์
  • ผู้ใช้อาจ "ยินยอม" โดยไม่รู้ตัว (มากับซอฟต์แวร์ฟรี)
  • ติดตามพฤติกรรมเพื่อแสดงโฆษณาตรงเป้า
  • สร้างความรำคาญ, ลดประสิทธิภาพเครื่อง

ความแตกต่างสำคัญ: "ความยินยอม" - Spyware ทำงานโดยไม่รู้/ไม่ยินยอม, Adware อาจถือว่าได้รับความยินยอม

ข้อ 10 - Chapter 4
อธิบายวงจรการตอบสนองเหตุการณ์ (Cyber Incident Response Cycle)

4 ขั้นตอนหลักตาม NIST SP 800-61:

1. Preparation (การเตรียมความพร้อม):

  • วางแผนก่อนเกิดเหตุการณ์
  • จัดตั้งทีม, นโยบาย, เครื่องมือ

2. Detection & Analysis (การตรวจจับและวิเคราะห์):

  • เฝ้าระวังและตรวจจับสัญญาณ
  • วิเคราะห์ยืนยัน, ประเมินผลกระทบ

3. Containment, Eradication & Recovery:

  • Containment: จำกัดความเสียหาย
  • Eradication: กำจัดต้นตอปัญหา
  • Recovery: กู้คืนระบบ

4. Post-incident Activity (กิจกรรมหลังเกิดเหตุ):

  • จัดทำรายงานสรุป
  • Lessons Learned
  • พัฒนากระบวนการ

ข้อ 11 - Chapter 4
องค์ประกอบการจัดการบันทึกเหตุการณ์ (Log Management) มีอะไรบ้าง?

4 องค์ประกอบหลักของ Log Management:

1. Log Generation (การสร้าง Log):

  • กำหนดระบบ/อุปกรณ์ที่ต้องบันทึก
  • ระบุข้อมูลที่ต้องบันทึก

2. Log Transmission (การส่ง Log):

  • กำหนดวิธีการส่งข้อมูล
  • รวบรวมไว้ที่ Centralized Log Server
  • ป้องกันการสูญหาย/แก้ไขระหว่างทาง

3. Log Storage and Disposal (การจัดเก็บและทำลาย):

  • กำหนดระยะเวลาเก็บรักษา
  • รักษา Confidentiality และ Integrity
  • กระบวนการทำลายเมื่อครบกำหนด

4. Log Analysis (การวิเคราะห์ Log):

  • กำหนดความถี่การตรวจสอบ
  • ตรวจหาสิ่งผิดปกติ
  • กำหนดสิทธิ์การเข้าถึง

ข้อ 12 - Chapter 5
3 ฟังก์ชันหลักของ IPS พร้อมตัวอย่างการทำงาน

1. Stops the attack itself (หยุดการโจมตีโดยตรง):

  • ยับยั้งการโจมตีทันที
  • ตัวอย่าง: ตรวจพบมัลแวร์ → ตัดการเชื่อมต่อเครือข่ายทันที

2. Changes the security environment (เปลี่ยนแปลงสภาพแวดล้อม):

  • ปรับการตั้งค่าอุปกรณ์ความปลอดภัย
  • ตัวอย่าง: ตรวจพบ Port Scanning → สั่ง Firewall บล็อก IP นั้น

3. Changes the attack's content (เปลี่ยนแปลงเนื้อหาการโจมตี):

  • แก้ไข/ลบส่วนที่เป็นอันตราย
  • ตัวอย่าง: อีเมลติดไวรัส → ลบไฟล์แนบ, ส่งต่อเฉพาะเนื้อหาปลอดภัย

ข้อ 13 - Chapter 5
วิธีการตรวจจับ Detection Methodologies ทั้ง 3 ลักษณะ

1. Signature-based Detection (ตามลายเซ็น):

  • เปรียบเทียบกับฐานข้อมูล Signatures
  • ✅ ประสิทธิภาพสูงกับภัยคุกคามที่รู้จัก
  • ❌ ไม่สามารถตรวจจับการโจมตีใหม่

2. Anomaly-based Detection (ความผิดปกติ):

  • สร้างโปรไฟล์พฤติกรรม "ปกติ"
  • ✅ ตรวจจับ Zero-day attacks ได้
  • ❌ อาจเกิด False Positives สูง

3. Stateful Protocol Analysis (วิเคราะห์สถานะโปรโตคอล):

  • ตรวจสอบตามมาตรฐานโปรโตคอล
  • ✅ แม่นยำกับการโจมตีที่ใช้ช่องโหว่โปรโตคอล
  • ❌ ตรวจไม่พบการโจมตีที่ไม่ละเมิดกฎ

ข้อ 14 - Chapter 5
องค์ประกอบของ IDPS มีอะไรบ้าง?

4 องค์ประกอบหลักของ IDPS:

1. Sensor or Agent:

  • Sensor: ใช้กับ Network-based IDPS - ดักจับแพ็กเก็ต
  • Agent: ติดตั้งบน Host-based IDPS - ตรวจสอบกิจกรรมในเครื่อง

2. Management Server:

  • ศูนย์กลางรับข้อมูลจาก Sensor/Agent
  • ทำ Correlation เชื่อมโยงเหตุการณ์
  • ระบุการโจมตีที่ซับซ้อน

3. Database Server:

  • จัดเก็บ Event Information
  • ใช้วิเคราะห์ย้อนหลัง
  • ทำ Forensic

4. Console:

  • User Interface สำหรับผู้ดูแลระบบ
  • ดูการแจ้งเตือน, วิเคราะห์เหตุการณ์
  • ตั้งค่าระบบ, สร้างรายงาน
