Chapter 1: Introduction to Big Data Analytics in Cybersecurity

Data Analytics และ Data Science เป็นสองศาสตร์ที่ทำงานกับข้อมูลเหมือนกัน แต่มีเป้าหมายและขอบเขตของงานที่แตกต่างกันอย่างชัดเจนครับ

  • Data Analytics (การวิเคราะห์ข้อมูล) จะมุ่งเน้นไปที่การนำข้อมูลที่มีอยู่มาวิเคราะห์เพื่อตอบคำถามทางธุรกิจที่เกิดขึ้นแล้ว หรือเพื่อค้นหาข้อมูลเชิงลึก (Insight) ที่ซ่อนอยู่ เปรียบเสมือนการนำข้อมูลในอดีตมาสรุปผลและทำความเข้าใจสถานการณ์ปัจจุบัน นักวิเคราะห์ข้อมูล (Data Analyst) มักจะทำงานกับข้อมูลที่มีโครงสร้าง (Structured Data) และใช้เครื่องมืออย่างสถิติและการสร้างภาพ (Data Visualization) เพื่อสื่อสารผลลัพธ์ให้คนอื่นเข้าใจได้ง่าย และนำไปใช้ตัดสินใจทางธุรกิจได้ทันที
  • Data Science (วิทยาศาสตร์ข้อมูล) มีขอบเขตที่กว้างกว่ามาก โดยจะครอบคลุมกระบวนการทั้งหมดตั้งแต่การตั้งคำถามใหม่ๆ ที่ยังไม่มีใครรู้คำตอบ ไปจนถึงการรวบรวมข้อมูลจากหลากหลายแหล่ง (ทั้งที่มีโครงสร้างและไม่มีโครงสร้าง) การเตรียมข้อมูล และที่สำคัญคือการสร้างแบบจำลอง (Model) เชิงพยากรณ์โดยใช้เทคนิคขั้นสูงอย่าง Machine Learning เพื่อทำนายสิ่งที่จะเกิดขึ้นในอนาคต หรือสร้างเครื่องมือใหม่ๆ เพื่อแก้ปัญหาทางธุรกิจที่ซับซ้อน

ความแตกต่างที่สำคัญ: ถ้าเปรียบเทียบง่ายๆ Data Analytics คือการใช้ข้อมูลเพื่ออธิบายสิ่งที่เกิดขึ้นและทำความเข้าใจปัจจุบัน ในขณะที่ Data Science คือการใช้ข้อมูลเพื่อสร้างแบบจำลองในการทำนายอนาคตและสร้างโซลูชันใหม่ๆ ครับ

การวิเคราะห์ข้อมูลสามารถแบ่งออกได้เป็น 4 ระดับ ซึ่งเปรียบเสมือนขั้นบันไดของการทำความเข้าใจข้อมูล จากง่ายไปสู่ซับซ้อน ดังนี้ครับ

  1. Descriptive Analytics (การวิเคราะห์เชิงพรรณนา): เป็นขั้นพื้นฐานที่สุด ตอบคำถามว่า "เกิดอะไรขึ้น?" (What happened?) คือการสรุปข้อมูลในอดีตให้ออกมาในรูปแบบที่เข้าใจง่าย เช่น การทำรายงานยอดขายประจำเดือน หรือการสร้างแดชบอร์ดเพื่อดูภาพรวมของข้อมูลที่เกิดขึ้น
  2. Diagnostic Analytics (การวิเคราะห์เชิงวินิจฉัย): เป็นขั้นที่ลึกลงไปอีกขั้นเพื่อตอบคำถามว่า "ทำไมสิ่งนั้นถึงเกิดขึ้น?" (Why did it happen?) เป็นการสืบค้นหาสาเหตุของผลลัพธ์ที่เกิดขึ้นในขั้น Descriptive เช่น การวิเคราะห์ว่าทำไมยอดขายในไตรมาสที่แล้วถึงลดลง โดยอาจพบว่าเกิดจากแคมเปญการตลาดที่ไม่มีประสิทธิภาพ
  3. Predictive Analytics (การวิเคราะห์เชิงพยากรณ์): เป็นการมองไปข้างหน้าเพื่อตอบคำถามว่า "จะเกิดอะไรขึ้นต่อไป?" (What will happen?) โดยใช้ข้อมูลในอดีตมาสร้างแบบจำลองทางสถิติหรือ Machine Learning เพื่อทำนายแนวโน้มในอนาคต เช่น การพยากรณ์ยอดขายในเดือนถัดไป หรือการทำนายว่าลูกค้าคนไหนมีแนวโน้มจะเลิกใช้บริการ
  4. Prescriptive Analytics (การวิเคราะห์เชิงให้คำแนะนำ): เป็นขั้นสูงสุดและซับซ้อนที่สุด ตอบคำถามว่า "เราควรทำอะไรดี?" (What should we do?) โดยระบบไม่เพียงแค่ทำนายอนาคต แต่ยังสามารถให้คำแนะนำทางเลือกที่ดีที่สุดเพื่อให้ได้ผลลัพธ์ที่ต้องการ เช่น ระบบอาจแนะนำว่าควรเสนอโปรโมชันส่วนลด 15% ให้กับลูกค้ากลุ่มที่มีแนวโน้มจะเลิกใช้บริการ เพื่อรักษาลูกค้ากลุ่มนี้ไว้

Big Data หรือข้อมูลขนาดใหญ่ มีคุณลักษณะสำคัญ 5 ประการ ที่ทำให้การจัดการและการวิเคราะห์มีความท้าทายและแตกต่างจากข้อมูลทั่วไป หรือที่เรียกว่า "5V's" ดังนี้ครับ

  1. Volume (ปริมาณ): คือ ขนาดของข้อมูลที่มีจำนวนมหาศาล ตั้งแต่ระดับเทราไบต์ (Terabytes) ไปจนถึงเซตตาไบต์ (Zettabytes) ซึ่งระบบคอมพิวเตอร์แบบดั้งเดิมไม่สามารถจัดเก็บหรือประมวลผลได้อย่างมีประสิทธิภาพ
  2. Velocity (ความเร็ว): คือ อัตราเร็วในการสร้างและไหลเข้าของข้อมูล ซึ่งเกิดขึ้นอย่างรวดเร็วและต่อเนื่องแบบเรียลไม์ เช่น ข้อมูลจากโซเชียลมีเดีย หรือข้อมูลจากเซ็นเซอร์ IoT ทำให้ต้องมีเทคโนโลยีที่สามารถประมวลผลข้อมูลได้อย่างทันท่วงที
  3. Variety (ความหลากหลาย): คือ รูปแบบของข้อมูลที่มีความหลากหลายสูงมาก ไม่ได้มีแค่ข้อมูลที่มีโครงสร้างชัดเจน (Structured Data) แบบตาราง แต่ยังรวมถึงข้อมูลกึ่งมีโครงสร้าง (Semi-structured) เช่น ไฟล์ XML และข้อมูลที่ไม่มีโครงสร้าง (Unstructured Data) เช่น ข้อความ, รูปภาพ, วิดีโอ, และไฟล์เสียง
  4. Veracity (ความถูกต้องน่าเชื่อถือ): คือ คุณภาพและความน่าเชื่อถือของข้อมูล เนื่องจากข้อมูลขนาดใหญ่มักมาจากหลายแหล่ง จึงอาจมีความไม่แน่นอน, ความไม่สมบูรณ์, หรือความขัดแย้งกันเองอยู่ ซึ่งเป็นความท้าทายในการคัดกรองและยืนยันความถูกต้องก่อนนำไปใช้งาน
  5. Value (คุณค่า): คือ คุณค่าที่ซ่อนอยู่ในข้อมูล ซึ่งเป็นเป้าหมายสูงสุดของการทำ Big Data Analytics โดยข้อมูลมหาศาลเหล่านี้จะต้องสามารถนำมาวิเคราะห์เพื่อสร้างประโยชน์ทางธุรกิจได้จริง เช่น การเข้าใจพฤติกรรมลูกค้า, การพัฒนานวัตกรรม, หรือการตัดสินใจที่แม่นยำขึ้น

Chapter 2: Cybersecurity Framework & Cyber Attack Framework

NIST Cybersecurity Framework (CSF) 2.0 เป็นกรอบการทำงานที่ช่วยให้องค์กรบริหารจัดการความเสี่ยงด้านความมั่นคงปลอดภัยไซเบอร์ได้อย่างเป็นระบบ ประกอบด้วย 6 ฟังก์ชันหลัก ซึ่งเปรียบเสมือนเสาหลักของโปรแกรมความปลอดภัยขององค์กร ดังนี้ครับ

  1. Govern (การกำกับดูแล): เป็นฟังก์ชันใหม่ที่เพิ่มเข้ามาในเวอร์ชัน 2.0 และมีความสำคัญอย่างยิ่ง ทำหน้าที่เป็นศูนย์กลางในการกำหนดทิศทาง, กลยุทธ์, และนโยบายด้านความปลอดภัยไซเบอร์ขององค์กรให้สอดคล้องกับเป้าหมายทางธุรกิจโดยรวม และเพื่อให้มั่นใจว่ามีการบริหารจัดการความเสี่ยงอย่างเหมาะสม
  2. Identify (การระบุ): คือการทำความเข้าใจในสินทรัพย์ (Assets) ที่องค์กรมี เช่น ข้อมูล, ระบบ, อุปกรณ์ต่างๆ และประเมินความเสี่ยงที่อาจเกิดขึ้นกับสินทรัพย์เหล่านั้น เพื่อให้รู้ว่าองค์กรจำเป็นต้องปกป้องอะไร และมีความเสี่ยงอะไรบ้าง
  3. Protect (การป้องกัน): คือการนำมาตรการควบคุมต่างๆ มาใช้เพื่อป้องกันสินทรัพย์ที่สำคัญตามที่ระบุไว้ในขั้น Identify เช่น การควบคุมการเข้าถึง, การฝึกอบรมพนักงาน, การรักษาความปลอดภัยของข้อมูล เพื่อลดโอกาสในการเกิดเหตุการณ์ไม่พึงประสงค์
  4. Detect (การตรวจจับ): คือการติดตั้งระบบเฝ้าระวังและกระบวนการต่างๆ เพื่อให้สามารถตรวจจับกิจกรรมที่น่าสงสัยหรือการโจมตีทางไซเบอร์ที่อาจเกิดขึ้นได้อย่างรวดเร็วที่สุด
  5. Respond (การตอบสนอง): คือการวางแผนและเตรียมกระบวนการเพื่อตอบสนองต่อเหตุการณ์ที่ตรวจพบ เช่น การจำกัดความเสียหาย, การวิเคราะห์เหตุการณ์, และการสื่อสารกับผู้ที่เกี่ยวข้อง เพื่อให้สามารถจัดการกับสถานการณ์ได้อย่างมีประสิทธิภาพ
  6. Recover (การฟื้นฟู): คือการวางแผนและเตรียมการเพื่อกู้คืนระบบและการดำเนินงานให้กลับสู่ภาวะปกติโดยเร็วที่สุดหลังจากเกิดเหตุการณ์ และลดผลกระทบต่อธุรกิจให้น้อยที่สุด

การสร้างและใช้งาน CSF Organizational Profile เป็นกระบวนการที่ช่วยให้องค์กรสามารถนำ Framework ไปปรับใช้ให้เข้ากับบริบทของตนเองได้ ซึ่งเป็นวงจรของการพัฒนาอย่างต่อเนื่อง มี 5 ขั้นตอนหลักดังนี้ครับ

  1. Scope the Organizational Profile (กำหนดขอบเขต): เริ่มต้นด้วยการตัดสินใจว่าเราจะสร้าง Profile นี้เพื่อใคร จะใช้กับทั้งองค์กร, แผนกใดแผนกหนึ่ง, หรือเฉพาะระบบงานที่สำคัญ เพื่อให้การดำเนินการมีเป้าหมายที่ชัดเจน
  2. Gather needed information (รวบรวมข้อมูลที่จำเป็น): รวบรวมข้อมูลเกี่ยวกับสถานะปัจจุบันขององค์กร ทั้งนโยบาย, กระบวนการ, และเทคโนโลยีด้านความปลอดภัยไซเบอร์ที่ใช้งานอยู่ เพื่อทำความเข้าใจว่าตอนนี้เรายืนอยู่จุดไหน
  3. Create the Organizational Profile (สร้าง Profile): นำข้อมูลที่รวบรวมได้มาสร้าง "Profile สถานะปัจจุบัน" (Current State Profile) และกำหนด "Profile สถานะเป้าหมาย" (Target State Profile) ที่องค์กรต้องการจะเป็นในอนาคต โดยอ้างอิงตามหมวดหมู่ต่างๆ ใน Framework
  4. Analyze gaps and create an action plan (วิเคราะห์ช่องว่างและสร้างแผนปฏิบัติการ): เปรียบเทียบระหว่าง Profile ปัจจุบันและเป้าหมาย เพื่อมองหา "ช่องว่าง" (Gaps) หรือจุดที่ต้องพัฒนา จากนั้นจัดลำดับความสำคัญและสร้างแผนปฏิบัติการ (Action Plan) ที่ชัดเจนว่าจะต้องทำอะไร, อย่างไร, และเมื่อไหร่ เพื่อไปให้ถึงเป้าหมาย
  5. Implement action plan and update Profile (ดำเนินการตามแผนและปรับปรุง Profile): นำแผนปฏิบัติการที่วางไว้ไปปฏิบัติจริง และที่สำคัญคือต้องมีการทบทวนและปรับปรุง Profile อยู่เสมอ เพื่อให้สอดคล้องกับการเปลี่ยนแปลงทางธุรกิจและภัยคุกคามที่เกิดขึ้นใหม่ๆ อยู่ตลอดเวลา

Cyber Kill Chain framework คือแบบจำลองที่อธิบายลำดับขั้นตอนการโจมตีทางไซเบอร์จากมุมมองของผู้โจมตี เปรียบเสมือน "ห่วงโซ่" ที่หากเราสามารถทำลายข้อใดข้อหนึ่งได้ ก็จะสามารถหยุดยั้งการโจมตีทั้งหมดได้ ประกอบด้วย 7 ขั้นตอนดังนี้ครับ

  1. Reconnaissance (การลาดตระเวน): เป็นขั้นตอนแรกที่ผู้โจมตีจะสอดแนมและรวบรวมข้อมูลเกี่ยวกับเป้าหมายให้ได้มากที่สุด เช่น โครงสร้างองค์กร, อีเมลพนักงาน, เทคโนโลยีที่ใช้ เพื่อค้นหาจุดอ่อนและวางแผนการโจมตี
  2. Weaponization (การสร้างอาวุธ): หลังจากได้ข้อมูลแล้ว ผู้โจมตีจะสร้าง "อาวุธไซเบอร์" ซึ่งก็คือมัลแวร์ (Malware) ที่ออกแบบมาโดยเฉพาะเพื่อใช้ประโยชน์จากช่องโหว่ที่พบ เช่น การสร้างไฟล์เอกสารที่มีโค้ดอันตรายฝังอยู่
  3. Delivery (การนำส่ง): ผู้โจมตีจะส่งอาวุธที่สร้างขึ้นไปยังเป้าหมาย ผ่านช่องทางต่างๆ ที่คาดว่าจะได้ผลที่สุด เช่น การส่งอีเมลหลอกลวง (Phishing), การล่อให้คลิกลิงก์บนเว็บไซต์, หรือการใช้ USB Drive ที่ติดไวรัส
  4. Exploitation (การใช้ช่องโหว่โจมตี): เมื่อเหยื่อหลงกลและเปิดไฟล์หรือคลิกลิงก์ "อาวุธ" ที่ส่งมาจะเริ่มทำงาน โดยใช้ประโยชน์จากช่องโหว่ (Vulnerability) ของซอฟต์แวร์ในเครื่องของเหยื่อเพื่อแทรกซึมและรันโค้ดอันตราย
  5. Installation (การติดตั้ง): เมื่อเจาะระบบได้แล้ว มัลแวร์จะทำการติดตั้งตัวเองลงในระบบของเหยื่อ เพื่อให้มันสามารถทำงานต่อไปได้เรื่อยๆ และยากต่อการถูกตรวจจับหรือลบออก
  6. Command & Control (C2) (การสั่งการและควบคุม): มัลแวร์ที่ติดตั้งแล้วจะทำการเชื่อมต่อกลับไปยังเซิร์ฟเวอร์ของผู้โจมตี เพื่อสร้างช่องทางลับสำหรับการควบคุมจากระยะไกล ทำให้ผู้โจมตีสามารถสั่งการเครื่องของเหยื่อได้ตามต้องการ
  7. Actions on Objectives (การดำเนินการตามเป้าหมาย): ในขั้นตอนสุดท้ายนี้ ผู้โจมตีจะใช้ช่องทาง C2 ที่สร้างขึ้นเพื่อลงมือทำตามเป้าหมายที่วางไว้ เช่น ขโมยข้อมูลสำคัญ, ทำลายระบบ, หรือเข้ารหัสไฟล์เพื่อเรียกค่าไถ่

Chapter 3: Cybersecurity Threat and Risks and Awareness Information for Cybersecurity

แม้ว่าสองคำนี้มักจะถูกใช้สลับกัน แต่แท้จริงแล้ว Information Security และ Cybersecurity มีความหมายและขอบเขตที่แตกต่างกันครับ

  • Information Security (ความมั่นคงปลอดภัยสารสนเทศ) มีขอบเขตที่ กว้างกว่า โดยมุ่งเน้นการปกป้อง "ข้อมูล" (Information) ในทุกรูปแบบ ไม่ว่าข้อมูลนั้นจะอยู่ในรูปแบบดิจิทัลบนคอมพิวเตอร์ หรืออยู่ในรูปแบบกายภาพ เช่น เอกสารที่พิมพ์บนกระดาษ หรือแม้แต่ข้อมูลในความคิดของคน โดยมีเป้าหมายหลักเพื่อรักษาคุณสมบัติ 3 ประการของข้อมูล คือ ความลับ (Confidentiality), ความถูกต้องครบถ้วน (Integrity), และความพร้อมใช้งาน (Availability) หรือ CIA Triad
  • Cybersecurity (ความมั่นคงปลอดภัยไซเบอร์) เป็น ส่วนหนึ่ง ของ Information Security ที่มุ่งเน้นการปกป้องข้อมูลและสินทรัพย์ใน โลกดิจิทัลหรือโลกไซเบอร์ โดยเฉพาะ ซึ่งหมายถึงการป้องกันระบบคอมพิวเตอร์, เครือข่าย, อุปกรณ์ และโปรแกรมต่างๆ จากภัยคุกคามและการโจมตีทางไซเบอร์

ความแตกต่างที่สำคัญ: ให้ลองนึกภาพว่า Information Security คือการปกป้อง "ห้องสมุด" ทั้งหมด ซึ่งรวมถึงหนังสือกระดาษ, ไมโครฟิล์ม และระบบคอมพิวเตอร์ ในขณะที่ Cybersecurity คือการปกป้อง "ระบบคอมพิวเตอร์และเครือข่าย" ของห้องสมุด จากแฮกเกอร์และไวรัสโดยเฉพาะครับ

อ้างอิง: Ch3_Threat - Risks and Awareness-2025.pdf (หน้า 4-8)

Deepfake Attack คือ การโจมตีรูปแบบหนึ่งที่ใช้เทคโนโลยีปัญญาประดิษฐ์ (AI) ขั้นสูงในการสร้างสื่อสังเคราะห์ปลอม ไม่ว่าจะเป็นวิดีโอ, ภาพ หรือเสียง ที่สามารถเลียนแบบใบหน้าและน้ำเสียงของบุคคลเป้าหมายได้อย่างแนบเนียนและสมจริงอย่างน่าทึ่ง จุดประสงค์หลักคือเพื่อใช้หลอกลวงให้เหยื่อหลงเชื่อว่าเป็นบุคคลนั้นจริงๆ แล้วนำไปสู่การฉ้อโกง, สร้างความเสียหายต่อชื่อเสียง, หรือเผยแพร่ข้อมูลเท็จ

ตัวอย่างการโจมตี: มีกรณีที่มิจฉาชีพใช้เทคโนโลยี Deepfake สร้างวิดีโอปลอมเป็น ปลัดกระทรวงมหาดไทย แล้วโทรศัพท์ไปหานายอำเภอ โดยเลียนแบบทั้งภาพและเสียงได้อย่างสมจริง เพื่อหลอกให้นายอำเภอช่วยโอนเงินเกือบแสนบาท โดยอ้างเหตุผลความจำเป็นเร่งด่วนต่างๆ ซึ่งเป็นตัวอย่างที่แสดงให้เห็นถึงความอันตรายของเทคโนโลยีนี้ที่สามารถปลอมเป็นบุคคลที่น่าเชื่อถือเพื่อหลอกลวงได้

แนวทางการป้องกัน: เนื่องจาก Deepfake มีความสมจริงสูง การป้องกันจึงต้องอาศัยความรอบคอบและมีสติเป็นอย่างมากครับ

  1. ตั้งสติและตั้งข้อสงสัย: หากได้รับการติดต่อจากบุคคลใดก็ตาม ไม่ว่าจะเป็นคนรู้จักหรือบุคคลสาธารณะ เพื่อขอข้อมูลส่วนตัวหรือขอให้ทำธุรกรรมทางการเงินอย่างเร่งด่วน ควรตั้งข้อสงสัยไว้ก่อนเสมอ
  2. สังเกตความผิดปกติ: แม้จะสมจริง แต่ Deepfake อาจยังมีจุดบกพร่องเล็กๆ น้อยๆ ให้สังเกตได้ เช่น การกะพริบตาที่ไม่เป็นธรรมชาติ, สีผิวหรือแสงเงาที่ผิดเพี้ยน, ริมฝีปากที่ขยับไม่ตรงกับเสียง หรือน้ำเสียงที่ราบเรียบไร้อารมณ์
  3. ตรวจสอบผ่านช่องทางอื่น (Verify): นี่คือวิธีที่ดีที่สุดครับ หากมีคนรู้จักติดต่อมาขอความช่วยเหลือ ควรวางสายแล้วโทรกลับไปที่เบอร์โทรศัพท์ของบุคคลนั้นที่เราบันทึกไว้เอง หรือติดต่อผ่านช่องทางอื่นที่มั่นใจได้ เพื่อยืนยันว่าเป็นตัวจริง ไม่ควรเชื่อการติดต่อที่เข้ามาหาเราเพียงฝ่ายเดียว
  4. ติดตามข่าวสาร: การรับรู้ข่าวสารเกี่ยวกับกลโกงและเทคโนโลยีใหม่ๆ ของมิจฉาชีพ จะช่วยให้เรารู้เท่าทันและไม่ตกเป็นเหยื่อได้ง่ายครับ

อ้างอิง: Ch3_Threat - Risks and Awareness-2025.pdf (หน้า 26-28)

Spyware และ Adware เป็นซอฟต์แวร์ไม่พึงประสงค์ทั้งคู่ แต่มีวัตถุประสงค์และวิธีการทำงานที่แตกต่างกันอย่างชัดเจนครับ

  • Spyware (สปายแวร์): เปรียบเสมือน "สายลับ" ที่แอบลักลอบเข้ามาในเครื่องคอมพิวเตอร์ของเรา โดยที่เราไม่รู้ตัวและไม่ยินยอม วัตถุประสงค์หลักของมันคือการสอดแนมและขโมยข้อมูลสำคัญ เช่น รหัสผ่าน, ข้อมูลบัตรเครดิต, หรือพฤติกรรมการใช้งานต่างๆ แล้วส่งกลับไปให้ผู้สร้าง ซึ่งถือเป็นภัยคุกคามที่อันตรายอย่างยิ่ง
  • Adware (แอดแวร์): เปรียบเสมือน "พนักงานแจกใบปลิว" ที่น่ารำคาญ วัตถุประสงค์หลักของมันคือการแสดงโฆษณาป๊อปอัปต่างๆ บนเครื่องของเรา โดยมักจะแฝงตัวมากับโปรแกรมฟรีที่เราดาวน์โหลดมาติดตั้ง ซึ่งในขั้นตอนการติดตั้ง เราอาจจะเผลอ "ยินยอม" ให้มันติดตั้งลงในเครื่องโดยไม่รู้ตัว แม้จะไม่ขโมยข้อมูลร้ายแรงเท่า Spyware แต่ก็สร้างความรำคาญและอาจทำให้เครื่องทำงานช้าลง

ความแตกต่างที่สำคัญ: อยู่ที่ "เจตนา" และ "การได้รับอนุญาต" ครับ Spyware มีเจตนาที่เป็นอันตราย (ขโมยข้อมูล) และทำงานโดยไม่ได้รับอนุญาต ในขณะที่ Adware มีเจตนาทางการค้า (แสดงโฆษณา) และมักจะอาศัยการอนุญาตแบบแฝงจากผู้ใช้ ครับ

Chapter 4: Incident Management and Cyber Resilience Framework & Log Management

วงจรการตอบสนองเหตุการณ์ทางไซเบอร์ หรือ Cyber Incident Response Cycle คือกระบวนการที่เป็นระบบและต่อเนื่องสำหรับจัดการกับเหตุการณ์ไม่พึงประสงค์ด้านความปลอดภัย ตั้งแต่การเตรียมตัวไปจนถึงการเรียนรู้เพื่อป้องกันเหตุในอนาคต ตามแนวทางของ NIST ประกอบด้วย 4 ขั้นตอนหลักดังนี้ครับ

  1. Preparation (การเตรียมความพร้อม): เป็นขั้นตอนเชิงรุกที่ทำ "ก่อน" เกิดเหตุ คือการวางแผนและเตรียมทรัพยากรให้พร้อมรับมือ เช่น การจัดตั้งทีมรับมือเหตุการณ์ (Incident Response Team), การเตรียมเครื่องมือที่จำเป็น, และการฝึกอบรมบุคลากร เพื่อให้แน่ใจว่าเมื่อเกิดเหตุการณ์ขึ้น ทุกคนจะรู้หน้าที่และสามารถจัดการได้อย่างรวดเร็ว
  2. Detection & Analysis (การตรวจจับและวิเคราะห์): เป็นขั้นตอนของการเฝ้าระวังเพื่อตรวจหาสัญญาณบ่งชี้ว่าอาจมีเหตุการณ์เกิดขึ้น เมื่อตรวจพบแล้ว จะต้องทำการวิเคราะห์เพื่อยืนยันว่าเป็นเหตุการณ์จริงหรือไม่, ประเมินความรุนแรงและขอบเขตของผลกระทบ เพื่อจัดลำดับความสำคัญในการตอบสนอง
  3. Containment, Eradication & Recovery (การจำกัดวง, การกำจัด, และการกู้คืน): เป็นขั้นตอนการลงมือปฏิบัติเพื่อควบคุมสถานการณ์ โดยเริ่มจาก Containment คือการจำกัดความเสียหายไม่ให้ลุกลาม, ต่อด้วย Eradication คือการกำจัดต้นตอของปัญหาให้หมดสิ้น เช่น การลบมัลแวร์, และสุดท้ายคือ Recovery คือการกู้คืนระบบที่ได้รับผลกระทบให้กลับมาทำงานเป็นปกติและปลอดภัยอีกครั้ง
  4. Post-incident Activity (กิจกรรมหลังเกิดเหตุ): เป็นขั้นตอนที่สำคัญมากแต่มักถูกมองข้าม คือการทบทวนและเรียนรู้จากเหตุการณ์ที่เพิ่งผ่านไป หรือ "Lessons Learned" เพื่อวิเคราะห์ว่ากระบวนการรับมือมีจุดไหนที่ทำได้ดี และมีจุดไหนที่ควรปรับปรุงแก้ไข เพื่อนำไปพัฒนากแผนการเตรียมความพร้อม (Preparation) ให้ดียิ่งขึ้น เป็นการปิดวงจรและเริ่มต้นใหม่เพื่อความแข็งแกร่งกว่าเดิม

อ้างอิง: Ch4_Incident Management and Cyber Resilience Framework _ Log.pdf (หน้า 13-14)

การจัดการบันทึกเหตุการณ์ หรือ Log Management คือกระบวนการบริหารจัดการข้อมูล Log ซึ่งเป็นบันทึกกิจกรรมต่างๆ ที่เกิดขึ้นในระบบคอมพิวเตอร์และเครือข่าย ถือเป็นหัวใจสำคัญของการตรวจจับและสืบสวนภัยคุกคามทางไซเบอร์ มีองค์ประกอบหลัก 4 ส่วนที่ทำงานต่อเนื่องกันเป็นวงจรชีวิตของ Log ดังนี้ครับ

  1. Log Generation (การสร้าง Log): เป็นจุดเริ่มต้น คือการกำหนดค่าให้อุปกรณ์และระบบต่างๆ (เช่น Firewall, Server, Application) ทำการบันทึกกิจกรรมที่สำคัญลงในไฟล์ Log โดยต้องกำหนดว่าจะบันทึกอะไรบ้าง และบันทึกในระดับความละเอียดแค่ไหน
  2. Log Transmission (การส่ง Log): เนื่องจาก Log ถูกสร้างขึ้นจากอุปกรณ์จำนวนมากที่กระจายอยู่ทั่วทั้งองค์กร ขั้นตอนนี้จึงเป็นการรวบรวม Log จากทุกแหล่งมาจัดเก็บไว้ที่ศูนย์กลาง (Centralized Log Management) อย่างปลอดภัย เพื่อให้ง่ายต่อการบริหารจัดการและป้องกันการถูกลบหรือแก้ไข
  3. Log Storage and Disposal (การจัดเก็บและทำลาย Log): เป็นการกำหนดนโยบายในการจัดเก็บ Log ที่รวบรวมมา เช่น จะเก็บไว้นานเท่าไหร่ (ตามกฎหมายหรือนโยบายองค์กร), จัดเก็บในรูปแบบใดเพื่อให้ค้นหาได้ง่ายและปลอดภัย, และมีกระบวนการทำลาย Log ที่ปลอดภัยเมื่อครบกำหนดเวลา
  4. Log Analysis (การวิเคราะห์ Log): เป็นขั้นตอนที่นำ Log มาใช้ให้เกิดประโยชน์สูงสุด คือการนำข้อมูล Log มาวิเคราะห์เพื่อค้นหาสิ่งผิดปกติ, รูปแบบการโจมตี, หรือร่องรอยของภัยคุกคาม ซึ่งอาจทำได้ทั้งแบบอัตโนมัติโดยใช้เครื่องมือ หรือการวิเคราะห์โดยผู้เชี่ยวชาญ

Chapter 5: Intrusion Detection and Prevention Systems

Intrusion Prevention System (IPS) หรือระบบป้องกันการบุกรุก เป็นระบบรักษาความปลอดภัยที่ไม่ได้ทำหน้าที่แค่ "ตรวจจับ" (Detect) เหมือน IDS แต่ยังสามารถ "ป้องกัน" (Prevent) ได้ด้วย โดยมี 3 ฟังก์ชันการทำงานหลักดังนี้ครับ

  1. หยุดการโจมตีโดยตรง (Stops the attack itself): นี่คือความสามารถหลักของ IPS ในการยับยั้งภัยคุกคามที่ตรวจพบได้แบบทันที
    • ตัวอย่าง: เมื่อมีข้อมูลวิ่งผ่าน IPS และระบบตรวจพบว่าข้อมูลนั้นมีลักษณะตรงกับ Signature ของไวรัสที่รู้จัก IPS จะทำการ "ทิ้ง" (Drop) แพ็กเก็ตข้อมูลนั้นทันที ไม่ปล่อยให้มันเดินทางต่อไปถึงเครื่องเป้าหมาย ทำให้การโจมตีล้มเหลวตั้งแต่เริ่มต้น
  2. เปลี่ยนแปลงสภาพแวดล้อมความปลอดภัย (Changes the security environment): IPS สามารถทำงานร่วมกับอุปกรณ์อื่นเพื่อปรับเปลี่ยนสภาพแวดล้อมให้การโจมตีทำได้ยากขึ้น
    • ตัวอย่าง: หาก IPS ตรวจพบว่ามี IP Address หนึ่งพยายามสแกนหาช่องโหว่ในเครือข่ายซ้ำๆ มันสามารถส่งคำสั่งไปแก้ไขกฎบนอุปกรณ์ Firewall โดยอัตโนมัติ ให้ทำการ "บล็อก" (Block) การเชื่อมต่อทุกชนิดจาก IP Address นั้นเป็นการชั่วคราว
  3. เปลี่ยนแปลงเนื้อหาของการโจมตี (Changes the attack's content): IPS บางชนิดมีความสามารถในการ "ทำความสะอาด" ข้อมูลที่เป็นอันตรายออกไป แล้วส่งต่อเฉพาะส่วนที่ปลอดภัย
    • ตัวอย่าง: เมื่อมีอีเมลหลอกลวง (Phishing) ที่มีลิงก์อันตรายแนบมาส่งผ่านเข้ามา IPS สามารถตรวจจับและ "ลบ" หรือ "แก้ไข" เฉพาะลิงก์ที่เป็นอันตรายนั้นออกจากเนื้อหาอีเมล แล้วจึงส่งอีเมลฉบับที่ปลอดภัยแล้วไปยังผู้รับปลายทาง

ระบบตรวจจับการบุกรุกมีเทคนิคในการ "มองหา" สิ่งผิดปกติอยู่ 3 วิธีหลัก ซึ่งแต่ละวิธีก็มีจุดเด่นและจุดด้อยแตกต่างกันไปครับ

  1. Signature-based Detection (การตรวจจับตามลายเซ็น): วิธีนี้เปรียบเสมือนการมี "แฟ้มประวัติอาชญากร" ระบบจะนำข้อมูลที่วิ่งผ่านเครือข่ายมาเปรียบเทียบกับฐานข้อมูลของรูปแบบการโจมตี (Signature) ที่เคยเกิดขึ้นแล้วและรู้จักกันดี หากมีข้อมูลส่วนใดตรงกับรูปแบบในฐานข้อมูล ระบบก็จะส่งเสียงเตือนทันที
    • จุดเด่น: แม่นยำมากในการตรวจจับภัยคุกคามที่รู้จักแล้ว และเกิดข้อผิดพลาด (False Positive) น้อย
    • จุดด้อย: ไม่สามารถตรวจจับการโจมตีรูปแบบใหม่ๆ หรือ Zero-day Attack ที่ยังไม่มีประวัติในแฟ้มได้เลย
  2. Anomaly-based Detection (การตรวจจับความผิดปกติ): วิธีนี้เปรียบเสมือนการ "เรียนรู้พฤติกรรมปกติ" ระบบจะใช้เวลาเรียนรู้และสร้างโปรไฟล์ว่าการใช้งานเครือข่ายในภาวะปกติเป็นอย่างไร (Baseline) จากนั้นจะคอยส่องอยู่ตลอดเวลา หากมีกิจกรรมใดที่ "ผิดปกติ" หรือเบี่ยงเบนไปจาก Baseline ที่เรียนรู้ไว้มากเกินไป ระบบก็จะแจ้งเตือนว่าอาจมีสิ่งผิดปกติเกิดขึ้น
    • จุดเด่น: สามารถตรวจจับการโจมตีรูปแบบใหม่ๆ ที่ไม่เคยรู้จักมาก่อนได้
    • จุดด้อย: มีโอกาสเกิดการแจ้งเตือนที่ผิดพลาด (False Positive) ได้สูง หากมีกิจกรรมใหม่ๆ ที่ไม่เคยทำมาก่อนแต่ไม่ใช่การโจมตีเกิดขึ้น
  3. Stateful Protocol Analysis Detection (การตรวจจับโดยวิเคราะห์สถานะของโปรโตคอล): วิธีนี้เปรียบเสมือนการมี "ผู้เชี่ยวชาญที่เข้าใจกฎกติกา" ของการสื่อสารในแต่ละโปรโตคอล (เช่น HTTP, FTP) เป็นอย่างดี ระบบจะคอยตรวจสอบว่าการสื่อสารนั้นเป็นไปตาม "กฎ" ที่ถูกต้องของโปรโตคอลหรือไม่ หากมีการกระทำที่ผิดกฎ เช่น การส่งคำสั่งในลำดับที่ผิดเพี้ยนไป ระบบก็จะถือว่าเป็นการโจมตี
    • จุดเด่น: มีความแม่นยำในการตรวจจับการโจมตีที่พยายามใช้ช่องโหว่ของโปรโตคอล
    • จุดด้อย: ไม่สามารถตรวจจับการโจมตีที่ไม่ได้ละเมิดกฎของโปรโตคอลได้ และต้องใช้ทรัพยากรในการประมวลผลสูง

ระบบตรวจจับและป้องกันการบุกรุก (IDPS) ที่สมบูรณ์นั้น ไม่ได้ทำงานด้วยตัวคนเดียว แต่ประกอบด้วยส่วนต่างๆ ที่ทำงานประสานกันเหมือนทีมรักษาความปลอดภัย ซึ่งมี 4 องค์ประกอบหลักดังนี้ครับ

  1. Sensor or Agent (เซ็นเซอร์ หรือ เอเจนต์): เปรียบเสมือน "ยาม" หรือ "กล้องวงจรปิด" ที่คอยสอดส่องและรวบรวมข้อมูลในจุดต่างๆ
    • Sensor: จะถูกวางไว้ในเครือข่ายเพื่อดักจับและตรวจสอบข้อมูล (Packet) ที่วิ่งผ่านไปมา
    • Agent: จะเป็นซอฟต์แวร์ที่ถูกติดตั้งไว้บนเครื่องคอมพิวเตอร์ (Host) เพื่อคอยตรวจสอบกิจกรรมที่เกิดขึ้นภายในเครื่องนั้นๆ
  2. Management Server (เซิร์ฟเวอร์จัดการ): เปรียบเสมือน "ห้องควบคุมกลาง" ที่รับข้อมูลทั้งหมดจากยามและกล้องวงจรปิด (Sensors/Agents) ทั่วทั้งองค์กรมาประมวลผลและวิเคราะห์ในภาพรวม ทำให้สามารถมองเห็นการโจมตีที่ซับซ้อนซึ่งอาจเกิดขึ้นจากหลายจุดพร้อมกันได้
  3. Database Server (เซิร์ฟเวอร์ฐานข้อมูล): เปรียบเสมือน "ห้องเก็บหลักฐาน" ทำหน้าที่จัดเก็บบันทึกเหตุการณ์ (Logs) ทั้งหมดที่ Management Server วิเคราะห์แล้วอย่างเป็นระบบ เพื่อใช้ในการตรวจสอบย้อนหลัง, การสืบสวนเชิงลึก (Forensics) หรือการออกรายงาน
  4. Console (คอนโซล): เปรียบเสมือน "จอภาพและแผงควบคุม" สำหรับผู้ดูแลระบบ (Admin) เป็นหน้าจอที่ใช้แสดงผลการแจ้งเตือนต่างๆ, ดูรายงาน, และใช้ในการปรับแต่งค่าการทำงานของระบบ IDPS ทั้งหมด
